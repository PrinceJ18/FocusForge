# ui-design.md

# FocusForge UI & Design System

Version: 1.0

---

# Purpose

This document defines the visual language and UI/UX principles of FocusForge.

Every AI-generated UI must follow this design system.

Consistency is more important than creativity.

Users should feel like every screen belongs to the same application.

---

# Design Philosophy

FocusForge is a modern SaaS application.

Design goals:

* Clean
* Premium
* Professional
* Minimal
* Productive
* Fast
* Consistent

Avoid unnecessary visual complexity.

The interface should help users focus.

---

# UI Principles

Every screen should be:

Simple

Intuitive

Accessible

Responsive

Consistent

Purposeful

Modern

Never add UI elements without a reason.

---

# Design Keywords

Think:

Linear

Notion

Stripe

Vercel

GitHub

Raycast

Modern SaaS dashboards

Avoid outdated dashboard styles.

---

# Color Philosophy

Dark-first interface.

Colors communicate meaning.

Use color intentionally.

Never rely on color alone to communicate important information.

# Background Hierarchy

Maintain clear depth.

Background

↓

Section

↓

Card

↓

Interactive Element

↓

Modal

↓

Tooltip

Users should immediately understand visual hierarchy.

---

# Spacing System

Follow an 8px spacing system.

Use consistent spacing throughout the application.

Avoid random margins.

Common spacing:

4

8

12

16

24

32

40

48

64

Consistency is more important than exact values.

---

# Border Radius

Maintain consistent corner radius.

Do not mix different radius values randomly.

Cards

Buttons

Inputs

Dialogs

Dropdowns

should feel like part of one design system.

---

# Typography

Hierarchy should be obvious.

Large headings

Section headings

Card titles

Body text

Captions

Labels

Avoid large blocks of text.

Prefer shorter paragraphs.

---

# Text Rules

Never use decorative fonts.

Prefer readability.

Keep text aligned consistently.

Avoid excessive bold text.

Use font weight to establish hierarchy.

---

# Icons

Use one icon library consistently.

Icons should support content,

not replace it.

Avoid decorative icons.

Use meaningful icons.

---

# Cards

Cards are the primary container.

Every card should have:

Clear title

Clear purpose

Consistent padding

Proper spacing

Balanced layout

Optional actions

Avoid crowded cards.

---

# Buttons

Primary button

One per major action.

Secondary button

Supporting actions.

Danger button

Delete

Reset

Remove

Critical actions only.

Avoid multiple competing primary buttons.

---

# Forms

Every form should include:

Labels

Validation

Helpful placeholders

Error messages

Loading state

Success feedback

Required indicators

Forms should be easy to complete.

---

# Inputs

Consistent height.

Consistent padding.

Visible focus state.

Clear labels.

Accessible contrast.

Validation feedback.

---

# Tables

Tables should:

Support sorting

Handle empty state

Remain responsive

Align content consistently

Avoid unnecessary columns

Readable on smaller screens

---

# Charts

Charts should communicate,

not decorate.

Always include:

Title

Legend

Labels

Meaningful colors

Readable axes

Avoid excessive visual effects.

---

# Dashboard Design

Dashboard should answer:

What matters most?

What requires attention?

What changed?

What should I do next?

The most important information belongs near the top.

---

# Navigation

Navigation should be predictable.

Users should never wonder where they are.

Active page should be obvious.

Navigation labels should be concise.

---

# Sidebar

Sidebar should contain:

Primary navigation only.

Avoid overcrowding.

Group similar features.

Maintain consistent ordering.

---

# Empty States

Every empty state should:

Explain why.

Suggest an action.

Encourage progress.

Never leave blank screens.

---

# Loading States

Every asynchronous operation requires:

Loading indicator

Skeleton loader where appropriate

Disabled actions

User feedback

Never freeze the interface.

---

# Error States

Errors should explain:

What happened.

Why it happened (if appropriate).

How the user can recover.

Avoid technical jargon.

---

# Animations

Animations should:

Guide attention.

Improve feedback.

Never distract.

Never delay productivity.

Keep animations short.

Prefer subtle motion.

---

# Responsive Design

Support:

Mobile

Tablet

Laptop

Desktop

Large Desktop

Never design for desktop only.

---

# Accessibility

Use semantic HTML.

Maintain sufficient contrast.

Support keyboard navigation.

Visible focus states.

Avoid relying only on color.

Use descriptive labels.

Accessibility is a requirement,

not an enhancement.

---

# Component Consistency

Components should look related.

Buttons

Cards

Dialogs

Inputs

Badges

Charts

Navigation

must follow the same visual language.

---

# Visual Hierarchy

Every page should clearly communicate:

Primary action

Secondary action

Important data

Supporting information

Users should scan the interface quickly.

---

# Microinteractions

Provide feedback for:

Hover

Click

Success

Failure

Loading

Completion

Interactions should feel responsive.

---

# Dark Theme

Dark mode is the primary experience.

Avoid:

Pure black backgrounds.

Pure white text.

High contrast fatigue.

Prefer comfortable long-term reading.

---

# AI UI Checklist

Before generating UI ask:

Is it consistent?

Is spacing correct?

Is typography balanced?

Is navigation clear?

Is accessibility respected?

Is it responsive?

Does it feel like modern SaaS?

Could this appear in a production application?

---

# UI Anti-Patterns

Avoid:

Inconsistent spacing

Random colors

Too many shadows

Too many gradients

Large paragraphs

Overcrowded screens

Nested cards

Too many buttons

Decorative animations

Low contrast

Inconsistent icons

Mixed design styles

---

# Definition of Great UI

Great UI is:

Simple

Elegant

Predictable

Fast

Accessible

Responsive

Professional

Consistent

Easy to understand

Pleasant to use

---

# Final Rule

Every new screen should look like it was designed by the same product team.

If a new feature visually stands out from the rest of the application, redesign it until it feels like a natural part of FocusForge.
